# Contributing guidelines

## Rules how we work

1. Pick an issue that is not on hold and assign yourself to it and add a "work in progress" label and change the state of this issue in GitHub project to "In Progress"
2. Try not to block issues. Take up an issue only when you are sure you may start straight away coding it
3. Implement and open a PR. We may avoid tests. The code does not have to be most performant and perfect, it has to be clear, easy to read, and working
4. Someone else will review the PR and merge it

## TIP

The easiest way to work is to use the dedicated GitHub Project for this repo and swap items from the backlog in the [Kanban view](https://github.com/users/Adam-it/projects/15/views/6)
